  • 6개 필드 필수/선택 입력
  • 닉네임: 영문/공백만 허용, 정규식 검증
  • 생년월일: YYYY.MM.DD 자동 포맷, 유효성 검증
  • 직업/직무 "기타" 선택 시 조건부 텍스트 입력
  • 변경점 감지 + 필수 검증 기반 저장 버튼 활성화
  • 완료 시 Toast + 이전 경로 복귀

4. 발견된 이슈

CRITICAL 레거시 NONE 값 마이그레이션 미처리

기존 성별 NONE 데이터 → 사용자에게 강제 재선택 UI를 띄워야 하는데 플로우 정의가 없음.

권장 수정: NONE 값이면 저장 버튼 차단 + "성별을 다시 선택해 주세요" 문구.

WARNING 직무(선택) → fieldOther 상태 초기화 누락

직무 선택/해제 시 fieldOther 텍스트가 남아있는 경우 있음.

권장 수정: field 변경 시 fieldOther도 리셋.

IMPROVEMENT 닉네임 중복 체크 로직 미명기

정규식만 있고 서버 중복 검사 호출 규칙이 스펙에 없음.

권장 수정: onBlur 디바운스 500ms 후 /profile/nickname/check 호출.
