취소 후 재예약 유도 (CancelRebookUX)

카테고리: 스케줄 적용 범위: 전체 대상: 정규(유료) 수업 취소자 — 연속 취소 예외 데이터: 취소 요청 이벤트 + 다음 예약 가능 슬롯
개요: 정규 수업 취소 직후 보강 예약을 유도하는 모달
수업 취소가 서버 확정된 직후 "수업 쿠폰 도착" 형태의 모달을 띄워 빠른 보강 예약을 유도합니다. 첫 정규 수업 취소에만 노출되며 반복 노출은 하지 않습니다.

1. 주요 기능

  • 취소 API 성공 콜백 직후 단 1회 노출
  • 이미 취소된 다음 수업은 트리거 제외
  • CTA "수업 예약하기" → 예약 화면 라우팅
  • 팝업 이탈/전환/예약완료 3경로 데이터 추적

3. 크로스레퍼런스 — 저장소별 구현 상태

프로토타입의 기획을 기준으로, 실제 저장소에 해당 기능이 구현되어 있는지 조사한 결과입니다.

저장소상태비고
Uphone3_front부분CenterPopup.tsx 모달 구조는 존재
uphone3_web부분LessonCancel API 연동은 있으나 유도 UI 없음
uphone3_api부분/lesson/cancel 엔드포인트 + 취소 플래그
uphone3_lms없음트리거 조건 관리 UI 없음
Uphone3_android없음WebView 래핑
uphone3_iOS없음WebView 래핑

4. 발견된 이슈

CRITICAL 트래킹 TODO 미구현

코드 내 "TODO: 트래킹 → 1) 유도 팝업 이탈" 주석이 남아있어, 효과 측정 자체가 불가능한 상태입니다.

권장 수정: GA4 또는 Amplitude로 popup_impression / popup_dismiss / popup_cta_click 3개 이벤트 발행.

WARNING imageUrl 로컬 경로 — 프리로드 미적용

모달 진입 순간 이미지 로딩으로 깜빡임(flicker) 발생 가능.

권장 수정: Link rel="preload" 또는 JS Image() 프리로드를 취소 API 호출 시점에 같이 트리거.

IMPROVEMENT 닫기 시 라우팅 정책 불명확

팝업 [X] 닫기 시 홈으로 갈지 스케줄 화면을 유지할지 정해지지 않음.

권장 수정: 기본은 현재 화면 유지 + "홈으로" 보조 링크 추가 권장.
