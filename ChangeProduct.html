상품 변경 (ChangeProduct)

카테고리: 결제 적용 범위: 전체 대상: 수강 중 상품 변경 희망자 데이터: 기존 주문 환불 예정 금액 + 신규 상품 + 결제수단
개요: 환불 후 재결제를 하나의 화면에서 처리하는 흐름
기존 상품을 중도 환불하고 동시에 신규 상품을 결제하여 상품을 교체합니다. 환불액과 신규 결제액을 하나의 화면에서 확인하고 결제합니다.

1. 주요 기능

  • 아코디언 3섹션: 기존 환불 / 신규 결제 / 결제수단
  • 수강 진행도 기반 환불액 자동 계산
  • 결제 수단 그리드 (카드/토스페이/네이버페이/계좌이체, 정기 상품은 정기 수단)
  • 만족 보장 상품 → "전액 환불" 배지 노출
  • 최종 결제액 4주 환산 표시

프로토타입의 기획을 기준으로, 실제 저장소에 해당 기능이 구현되어 있는지 조사한 결과입니다.

4. 발견된 이슈

CRITICAL dialog 모듈 mock 상태

프로토타입 내부에서 dialog가 mock으로 되어 있어 실제 utils/dialog 모듈과 연동 필요. 결제 컨펌/실패 다이얼로그가 실제 UX와 다르게 동작할 수 있습니다.

권장 수정: 실제 `utils/dialog.ts` 의 showConfirm / showAlert API로 교체.

WARNING 환불 수단 제약 — 원 결제수단만 가능?

API가 원 결제수단만 환불 허용하는지, 프론트에서도 제한해야 하는지 스펙에 명시 안 됨.

권장 수정: API 문서 확정 후, 프론트에서 해당 수단을 자동 선택 + disabled 처리.

IMPROVEMENT 결제 실패/취소 시나리오 처리 미흡

현재는 try-catch로 alert만 띄움.

권장 수정: 결제 실패 코드별 안내 문구(잔액부족/한도초과/세션만료 등)와 재시도 UX 추가.
