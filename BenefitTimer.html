혜택 마감 타이머 (BenefitTimer)

카테고리: 홈/배너 적용 범위: 한국·일본 (타임존 기반) 대상: B2C / B2B2C (개인결제·재수강 등 7가지 배너 케이스) 데이터: 배너 데이터 + 쿠폰/적립금 만료 시각
개요: 앱 배너 내 쿠폰/적립금 마감 카운트다운 타이머
배너 영역에 쿠폰·적립금 등 혜택의 만료 시점을 4단계(MM:SS / HH:MM:SS / 오늘까지 / M/D까지)로 표시해 긴박감을 부여하는 컴포넌트입니다.

1. 주요 기능

  • 만료 시각까지 남은 시간에 따른 4단계 표시
  • 혜택 미보유 시 타이머 자동 숨김(null 반환)
  • 배너 타입별 (new-custom / new-trial / renewal) 노출 정책
  • 한국·일본 타임존 기반 시각 계산

2. 데이터 모델

필드설명
BannerTypenew-custom / new-trial / renewal 등 7가지
hasCoupon / hasReward보유 여부 — 둘 다 false면 렌더 안 함
expiresAt만료 시각 (ISO / epoch)

3. 크로스레퍼런스 — 저장소별 구현 상태

프로토타입의 기획을 기준으로, 실제 저장소에 해당 기능이 구현되어 있는지 조사한 결과입니다.

저장소상태비고

uphone3_web없음신규 기능 — 구현 필요
uphone3_api부분/api/banner/list, PurchaseService 에 만료 계산 로직 일부
uphone3_lms없음관리자 화면 필요
Uphone3_android없음WebView 래핑만 — 별도 구현 없음

4. 발견된 이슈

CRITICAL Date.now() 클라이언트 시간 기준 — 서버 시간 불일치 위험

사용자 기기 시간이 틀리면 타이머 값이 잘못 표시되고, 만료 안 됐는데 혜택이 만료된 것처럼 보일 수 있습니다.

권장 수정: expiresAt 비교 시 서버 응답의 Date 헤더 또는 /api/time 같은 서버 시각 엔드포인트로 오프셋을 보정하세요.

WARNING 만료 이후 null 반환 → 빈 공간 노출

타이머가 사라지면 배너 높이가 줄어 레이아웃이 흔들립니다.

권장 수정: 만료 후 "혜택 종료" placeholder 또는 다음 배너로 전환되도록 fallback 렌더링을 넣으세요.

IMPROVEMENT 타이머 state 클래스명 일관성 (short/long/tomorrow/far)

SCSS/Tailwind 클래스가 단계별로 통일되지 않아 스타일 충돌 가능.

권장 수정: 상태 enum(urgent / today / tomorrow / future)으로 네이밍 통일.
