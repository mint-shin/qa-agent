무료체험 스마트 대기실 (TrialSmartWaiting)

카테고리: 체험 적용 범위: 한국, 일본 대상: B2C / B2B2C (주니어 제외) 데이터: 프로필 + 교재 + 수업 시간 + 강사
개요: 신청~수업 시작까지 5단계 위자드 경험
예약 직후부터 수업 시작까지 Setup(프로필) → Bridge(분석) → Prep(워밍업) → Home/Textbook(대기) → In-Class(수업)로 이어지는 5단계 통합 경험입니다.
목차
  • 1. 주요 기능
  • 2. 데이터 모델
  • 3. 크로스레퍼런스 (저장소별 구현 상태)
  • 4. 발견된 이슈
  • 5. 테스트 케이스
  • 6. 권장 로드맵

1. 주요 기능

  • Step 1 Setup: 학습목적/직업/프로필 3-Q 설문
  • Step 2 Bridge: 2~3초 분석 애니메이션
  • Step 3 Prep: Ice-breaking/Listening/Speaking/Questions 4카드
  • Step 3 Speaking: Azure 기반 4-metric 피드백 (정확도/유창성/완성도/운율)
  • Step 4 Home/Textbook: 카운트다운 + 교재 미리보기
  • Step 5 In-Class: 3단 세일즈 플로우 기반 수업

2. 데이터 모델

필드설명
Setupreason, job, nickname, gender, birthday
PrepselectedQuestion, listeningScore, speakingMetrics(4)
Warmup 결과강사LMS 전송

4. 발견된 이슈

WARNING Setup 프로필 "선택 안함" 정책 미명기

성별은 "선택 안함" 있는데 닉네임/생년월일 등은 없음. 레거시 NONE 값 마이그레이션 전략 부재.

권장 수정: 각 필드 별 필수/선택 여부와 기존 NONE 값 처리 스펙 추가.

IMPROVEMENT Bridge 애니메이션 스펙 부재

2~3초 동안 어떤 리소스를 preload 할지 명확하지 않음.

권장 수정: preload 대상(강사 아바타, 교재 이미지, 음원)을 문서화.

IMPROVEMENT Speaking 점수 77~79점 컬러 정의 누락

컬러 매핑 시 경계값 누락으로 미묘한 시각 불일치.

권장 수정: 컬러 매핑 table을 design-system 토큰으로 관리.
